Types of web workers

  • Decicated workers
  • Shared workers

Dedicated workers

  • The code must be contained in a separate file
  • Communication via message passing
  • Event listeners to catch messages between source and worker

Shared workers

  • Workers that can be used by multiple scripts running as long as they are in the same domain
  • Shared workers are more complicated than dedicated workers
  • Scripts must communicate via an active port

Use cases of web workers

  • Prefetching and/or caching data for later use
  • Code syntax highlighting or other real-time text formatting
  • Spell checker
  • Analyzing video or audio data
  • Background 1/0 or polling of web services
  • Processing large arrays or humungous JSON responses
  • Image filtering in canvas
  • Updating many rows of a local web database

Web workers limitations

  • Cannot directly manipulate the DOM
  • Has limited access to methods and properties of the window object
  • Cannot be run directly from the filesystem. It can only be run via a server
